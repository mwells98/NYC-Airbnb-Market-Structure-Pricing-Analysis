I started working on reducing the NaN values by starting with listings with 0 availability were removed to focus on active inventory. Including inactive listings could distort pricing distributions and host behavior patterns. Removing listings with zero availability reduced missing price values from 15,126 to 2,783, indicating most missing price entries were associated with inactive listings. Because 2,783 was small in comparison to the rest of the data I dropped all NaN values for the price column. Then I used logic to determine all host_is_superhost that are NaN are going to be False. Finally, for the bedrooms, bathrooms, and beds I filled the missing values with the median column value for each respected column transforming this data to display 0 NaN values.

# This block replaces all 0s with the list with the median value of that column grouped by room_type
for col in ['bedrooms', 'bathrooms', 'beds']:
    df[col] = df[col].replace(0, np.nan)
    df[col] = df.groupby('room_type')[col].transform(lambda x: x.fillna(x.median()))

Here I noticed something wrong with the data. While there might not be NaN values in the bedrooms, bathrooms, and beds columns, there are 0s and that doesn't make sense as all listings should have at least one room, one bathroom, and one bed. I replaced them with median imputation grouped by room type, preserving structural differences between categories while reducing distortion from extreme values. This approach assumes zero values reflect data entry issues rather than legitimate studio classifications.
